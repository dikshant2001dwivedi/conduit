# Copilot Session Bootstrap (Conduit Migration)

Before implementation, always run this startup protocol in order:

1. Read PROJECT-STATE.md
2. Read TASKS.md
3. Read MIGRATION-MAP.md
4. Read DOCUMENT-CHANGES.md
5. Read this file (.copilot-instructions.md)
6. Run:
	 - git log --oneline -5
	 - git status

If any core file is missing or stale, fix docs first and then continue.

## Project Scope
- Source repo: ../realWorld-DjangoRestFramework (read-only behavior baseline)
- Target repo: conduit (all migration code and migration docs)
- Migration type: incremental Django -> NestJS
- Task rule: complete exactly one atomic task at a time

## Guardrails
- Do not rely on chat history; rely on repository files.
- Do not start the next task until current task verification passes.
- Preserve documented Django bugs (BUG-001, BUG-002) until post-migration cleanup.
- Preserve documented status quirks (login 202, comment create 200).
- Use Prisma (not TypeORM).
- Use Authorization: Bearer <token> in NestJS implementation and contracts.

## Verification Rules
- Before each task:
	- cd ../realWorld-DjangoRestFramework
	- python manage.py test
- For contract capture in Phase 0, run contracts against BASE_URL=http://localhost:8000.
- For implementation phases, run contracts against BASE_URL=http://localhost:3000.
- Always run only the domain subset for the current task unless task requires full suite.

## Required Checkpoints Per Task
- Update PROJECT-STATE.md
- Run task verification commands from TASKS.md
- Commit with conventional format: type(scope): summary

## Session Wrap-Up
- Update PROJECT-STATE.md (phase, task, completed work, verification status, next tasks)
- Update DOCUMENT-CHANGES.md only when behavior/spec decisions change
- Commit and verify clean working tree with git status
